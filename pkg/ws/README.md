# pkg/ws

Пакет `ws` содержит транспортный каркас для WebSocket-соединений без чатовой бизнес-логики.

## Файлы

### `conn.go`

Определяет базовые абстракции транспорта:

- `Conn` - интерфейс WebSocket-соединения.
- `Upgrader` - интерфейс, который переводит HTTP-запрос в WebSocket-соединение.

Этот файл нужен, чтобы пакет не зависел от конкретной библиотеки вроде `gorilla/websocket`.

### `errors.go`

Содержит типовые ошибки пакета:

- отсутствие `upgrader` или `hub`
- пустой `client id` (`int64 == 0`)
- повторная подписка клиента
- попытка отправить сообщение в закрытое соединение
- переполненный send buffer

Нужен как единая точка для ошибок lifecycle WebSocket-клиента.

### `client.go`

Описывает WebSocket-клиента:

- хранит `id int64` клиента
- хранит `Conn`
- держит канал `send` для исходящих сообщений
- реализует `Send`
- реализует `WriteLoop`
- реализует `Close`

Это локальная модель одного активного WS-подключения.

### `hub.go`

Описывает хаб подключений:

- хранит подписанных клиентов
- реализует `Subscribe`
- реализует `Unsubscribe`
- позволяет получить клиента по `id int64`
- умеет делать `Broadcast`
- умеет закрывать все подключения через `Close`

Это общий runtime-слой для управления набором WS-клиентов.

### `serve.go`

Содержит HTTP-to-WS lifecycle:

- `ServeWS(...)` - создаёт HTTP handler для WebSocket endpoint
- `ServeOptions` - набор callback-ов и настроек
- `ClientIDFunc` - как определить `id int64` клиента из HTTP-запроса
- `OnConnect` - логика при подключении
- `OnMessage` - логика при входящем сообщении
- `OnClose` - логика при закрытии соединения
- `Group` - контейнер для нескольких `Hub`, например по комнатам

Этот файл связывает HTTP handler, upgrade соединения, чтение, запись и cleanup.

### `ws_test.go`

Содержит тесты пакета:

- подписка и отписка клиента
- broadcast сообщений
- закрытие клиента
- полный lifecycle через `ServeWS`

Нужен для проверки, что базовый runtime пакета работает корректно.

## Что должно оставаться вне `pkg/ws`

В этот пакет не стоит класть:

- комнаты чата
- права доступа
- формат доменных сообщений
- сохранение сообщений в БД
- unread counters
- бизнес-правила модерации

Это уже зона `internal/app/chat` или обработчиков gateway.
